# Choosing the Right Accessibility Tool

LambdaTest Accessibility offers several tools for different testing needs. Use this guide to pick the right starting point based on your workflow, team setup, and what you want to test.

## Use DevTools when

- you want manual or browser-assisted scanning
- you need quick investigation on a live page
- you want scan types like quick, full page, partial page, multi-page, or workflow scan

## Use Automation when

- you want accessibility checks in automated test execution
- you want reusable regression coverage
- you need framework-based reports from supported web automation stacks

## Use Test Scheduling when

- you want recurring site scans
- you need sitemap or crawler-driven scanning
- you want regular accessibility visibility without rerunning manual flows
- you want the Accessibility-native scheduling surface rather than Web Scanner

## Use App Scanner (Manual) when

- you are validating mobile app screens interactively
- you want to inspect issues screen by screen on real devices

## Use Native App Automation when

- you already run Appium automation
- you want mobile app accessibility reports from automated test runs

## Use Web Scanner when

- you are already working inside the Web Scanner product surface
- you want URL-based Accessibility scanning from that integrated workflow
- you do not need Accessibility DevTools or framework-based Automation to start the scan

## Use KaneAI when

- you are authoring a mobile test in KaneAI
- you want to insert Accessibility scan instructions directly into the authored execution flow
- you want KaneAI-driven execution rather than a direct Appium setup

## Use Screen Reader testing when

- you need to validate how your site or app behaves with assistive technology like NVDA, JAWS, VoiceOver, or TalkBack
- you are running manual screen reader validation sessions on LambdaTest real devices or VMs

## Use Checklists and Rule References when

- you want to understand which WCAG rules are checked by automated scans
- you need a manual checklist for the rules that automation does not cover
- you want to look up a specific rule and its remediation guidance

## Use Accessibility MCP Server when

- you want AI-assisted accessibility analysis on supported public or local app workflows
