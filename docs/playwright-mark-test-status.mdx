By default, the status of each successfully run test is marked as **Completed** on TestMu AI. Since assertions run client-side, you need to explicitly mark tests as **Passed** or **Failed** based on your testing requirements.

## Mark as Passed
---

```js
await page.evaluate(_ => {},
  `lambdatest_action: ${JSON.stringify({
    action: 'setTestStatus',
    arguments: { status: 'passed', remark: 'Title matched' }
  })}`
)
```

## Mark as Failed
---

```js
await page.evaluate(_ => {},
  `lambdatest_action: ${JSON.stringify({
    action: 'setTestStatus',
    arguments: { status: 'failed', remark: 'Title not matched' }
  })}`
)
```

<Tip>
Call these methods from an `afterEach` or `afterAll` hook where assertion results are available, so the status accurately reflects the test outcome.
</Tip>
