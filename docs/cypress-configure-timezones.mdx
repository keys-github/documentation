When testing websites that display different content based on the user's timezone, you can set a custom timezone for the remote machine running your Cypress tests. This affects how date and time functions behave in your application during testing.

## Using the CLI
---

```bash
lambdatest-cypress run --tz "US/Eastern"
```

| Flag | Purpose | Type |
|------|---------|------|
| `--tz, --timezone` | Set custom timezone on the test machine | String |

## Common Timezone Values
---

| Region | Timezone Value |
|--------|----------------|
| US Eastern | `US/Eastern` |
| US Pacific | `US/Pacific` |
| United Kingdom | `Europe/London` |
| India | `Asia/Kolkata` |
| Japan | `Asia/Tokyo` |
| Australia | `Australia/Sydney` |
| Germany | `Europe/Berlin` |
| Brazil | `America/Sao_Paulo` |
| Singapore | `Asia/Singapore` |
| UAE | `Asia/Dubai` |

<Tip>
To test location-based content like pricing or language preferences, you can also [simulate IP geolocation for Cypress tests](/support/docs/cypress-geolocation/).
</Tip>
