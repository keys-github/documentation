# Getting Started (Quick Setup)

Use this page when you want the fastest path from “I need to test accessibility” to “I have my first report.”

This guide helps you pick the right first workflow—whether manual web, automation, scheduling, Web Scanner, or mobile—and summarizes the minimum setup to reach your first scan and report. It also frames prerequisites and a sensible first-run sequence so you execute in the correct surface and can interpret your initial results.

## Choose your first path

- **Manual web testing**: start with [Accessibility DevTools](/support/docs/accessibility-devtools/).
- **Automated web testing**: start with [Accessibility Automation (Overview)](/support/docs/accessibility-automation/).
- **Scheduled web scans**: start with [Test Scheduling - Sitemap (Overview)](/support/docs/accessibility-test-scheduling/).
- **Web Scanner-based scans**: start with [Getting Started with Web Scanner](/support/docs/web-scanner-getting-started/).
- **Manual mobile app testing**: start with [Accessibility App Scanner (Overview)](/support/docs/accessibility-app-scanner/).
- **Mobile app automation**: start with [Native App Automation Appium (Overview)](/support/docs/accessibility-native-app-automation-test/).
- **KaneAI-authored mobile flows**: start with [Mobile App Accessibility Testing](/support/docs/kaneai-mobile-app-accessibility/).

## Minimum prerequisites

- access to the relevant Accessibility feature for your account
- a website, web app, or mobile app you are authorized to test
- the correct browser, device, framework, or scan surface for your use case

## First-run checklist

1. Pick the right product surface.
2. Confirm whether your flow is manual, automated, scheduled, or integrated through Web Scanner or KaneAI.
3. Run the scan.
4. Open the report.
5. Review issue counts, severity, and next actions.

## Common mistakes to avoid

- using DevTools when you actually need scheduled or automated execution
- using Web Scanner docs for the Accessibility-native scheduling surface
- using KaneAI guidance when your team is running direct Appium automation
