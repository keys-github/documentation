# Accessibility App Scanner (Overview)

Accessibility App Scanner is the manual mobile app workflow for reviewing Accessibility issues on real devices, screen by screen.

You will find guidance on when App Scanner is appropriate for validating mobile accessibility. The page also describes what the manual on-device flow entails and how it compares to Appium automation and KaneAI-authored testing.

## When to use this

Use App Scanner when you want to inspect Android or iOS app screens interactively and validate findings as you move through the app.

## Typical workflow

1. Upload or select the app.
2. Start the device session.
3. scan important screens manually.
4. review the issues.
5. save the report and move into dashboard analysis if needed.

## Product boundary

App Scanner is a manual mobile-app workflow. If you are running Appium code, use [Native App Automation Appium (Overview)](/support/docs/accessibility-native-app-automation-test/). If you are authoring the flow in KaneAI, use [Mobile App Accessibility Testing](/support/docs/kaneai-mobile-app-accessibility/).
