TestMu AI provides several debugging tools on the Automation Dashboard to help you investigate failed Cypress tests.

## Video Recording
---

Every Cypress test run is recorded as it executes on the remote machine. Use the video playback to retrace the steps leading to a test failure. Videos are available in the session details on the Automation Dashboard.

## Console Logs
---

Browser console output is captured for each test session. Check console logs for JavaScript errors, warnings, or unexpected output that may have caused the failure.

## Network Logs
---

Network logs capture HTTP requests, responses, and latency data. Enable them by setting `network` to `true`:

```json lambdatest-config.json
{
  "run_settings": {
    "network": true
  }
}
```

For advanced network log options, learn how to [capture network logs for Cypress tests](/support/docs/cypress-network-logs/).

## Command Logs
---

For detailed Cypress command-by-command logs showing each step of test execution, set up the `cypress-terminal-report` package. Learn how to [set up detailed command logs for Cypress](/support/docs/cypress-detailed-command-logs/).
