# Robot Framework

Use this page when your Accessibility Automation flow runs through Robot Framework.

This page places Robot Framework inside the Selenium-based Accessibility Automation model so you can see which layer owns browser keywords versus enabling accessibility in the underlying session. Refer to it when you run Robot suites and need to interpret reports consistently with other Selenium-backed frameworks.

## When to use this

Use this workflow when your team runs browser automation through Robot Framework and wants Accessibility checks in the same execution path.

## Prerequisites

- a Robot Framework web automation project
- a Selenium-backed session with Accessibility enabled
- access to the Accessibility dashboard for report review

## Typical workflow

1. enable Accessibility in the underlying Selenium session
2. run the Robot Framework suite
3. review the resulting report in the dashboard
4. use the report views for triage and follow-up
