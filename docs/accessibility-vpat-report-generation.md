# VPAT Report Generation

Accessibility testing can support VPAT-oriented workflows by supplying issue evidence, coverage summaries, and remediation context.

VPAT-style documentation still depends on human evaluation, yet structured accessibility reporting can accelerate evidence collection and audit prep conversations. This page clarifies which VPAT-oriented tasks the product’s reports support well and which steps remain firmly in human hands. Read it before you promise stakeholders that exports alone equal a finished VPAT.

## What Accessibility reports help with

- surfacing automated findings
- showing coverage summaries
- providing evidence for remediation planning
- supporting discussion during audit preparation

## What Accessibility reports do not replace

Accessibility reports do not replace the full human evaluation and documentation process required for final VPAT preparation.
